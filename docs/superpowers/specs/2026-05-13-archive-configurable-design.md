# 归档刮削可配置化设计

**日期**: 2026-05-13  
**状态**: 已确认

## 背景

归档模块当前把目录命名、文件命名、分类规则都写死在后端代码里。这样虽然能用，但以后想改结构时，必须改代码、发版，成本很高。

这次调整的目标，是把这些规则放到运行时配置里，让用户能在前端直接改。

## 本次范围

### 1. 命名模板

继续使用现有 4 个字段，不新增字段：

- `movie_root_dir`
- `tv_root_dir`
- `movie_filename_template`
- `tv_filename_template`

这次的关键变化是：`movie_filename_template` 和 `tv_filename_template` 不再只表示最后的文件名，而是允许写成“相对路径 + 文件名”。  
这样就能直接表达三级目录，甚至更深的目录。

例如：

```jinja2
{{ title }}{% if year %} ({{ year }}){% endif %}/{{ title }}{% if year %} ({{ year }}){% endif %}{{ ext }}
```

剧集示例：

```jinja2
{{ title }}{% if year %} ({{ year }}){% endif %}/第{{ season }}季/{{ title }} - {{ season_episode }}{{ ext }}
```

### 2. 分类规则

继续保留现有分类规则，还是负责前两层目录：

- 电影：`电影/华语电影`、`电影/日韩电影`、`电影/外语电影`
- 剧集：`剧集/纪录片`、`剧集/动漫`、`剧集/综艺`、`剧集/国产剧`、`剧集/日韩剧`、`剧集/美英剧`

也就是说：

- 分类规则决定“第 2 层目录”
- 模板决定“第 3 层及以后”

## 后端行为

- 归档时先算出分类目录
- 再渲染模板得到相对路径
- 后端检查路径安全，禁止模板输出跑出归档目录外
- 模板写错时，不让任务整体失败，回退到默认模板

## 前端行为

归档页继续提供：

- 5 个命名输入框
- 分类规则编辑
- 实时预览

预览目标是让用户一眼看出最终会生成什么路径，不需要自己脑补。

## 默认值

配置为空时，行为保持和现在一致，确保升级后不影响旧任务：

- 默认电影根目录：`电影`
- 默认剧集根目录：`剧集`
- 默认电影模板：`{{ title }}{% if year %} ({{ year }}){% endif %}/{{ title }}{% if year %} ({{ year }}){% endif %}{{ ext }}`
- 默认剧集模板：`{{ title }}{% if year %} ({{ year }}){% endif %}/第{{ season }}季/{{ title }} - {{ season_episode }}{{ ext }}`

## 错误处理

- 模板语法错了：前端保存时直接报错
- 模板渲染时变量缺失：后端记录 warning，并回退默认模板
- 分类规则为空：允许，表示不生成分类目录

## 验收标准

- 现有分类结果不变
- 现有默认路径结构不变
- 用户可以用现有字段拼出三级目录
- 前端可以实时看到预览
